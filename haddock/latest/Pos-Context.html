  • ncRecoveryHeader :: !(TMVar (NodeId, BlockHeader ssc))

    In case of recovery mode this variable holds the latest header hash we know about, and the node we're talking to, so we can do chained block requests. Invariant: this mvar is full iff we're more than recoveryHeadersMessage blocks deep relatively to some valid header and we're downloading blocks. Every time we get block that's more difficult than this one, we overwrite. Every time we process some blocks and fail or see that we've downloaded this header, we clean mvar.

  • ncProgressHeader :: !(TMVar (BlockHeader ssc))

    Header of the last block that was downloaded in retrieving queue. Is needed to show smooth prorgess on the frontend.
